## Getting Help

### Before Contacting Support

1. **Check Documentation** - Search our docs for answers
2. **Search Community** - Look for similar issues
3. **Try Troubleshooting** - Use our troubleshooting guide
4. **Test in Sandbox** - Verify in test environment

### Contacting Support

When contacting support, include:

- **Account Information** - Account email or ID
- **Issue Description** - Clear description of the problem
- **Steps to Reproduce** - How to reproduce the issue
- **Error Messages** - Full error messages and logs
- **Environment Details** - SDK version, environment, etc.

### Support Request Template

```
Subject: [Priority] Issue with user creation API

Account: user@example.com
Plan: Pro
Environment: Production

Issue Description:
Unable to create users via API, getting 400 error.

Steps to Reproduce:
1. Call POST /v1/users with valid data
2. Receive 400 Bad Request error
3. Error message: "Invalid request"

Request Details:
- SDK: @hackmamba/sdk v1.2.3
- Environment: Production
- API Key: sk_live_... (masked)

Error Logs:
[Include relevant error logs]

Expected Behavior:
User should be created successfully.

Actual Behavior:
Request fails with 400 error.
```

## Escalation Process

### When to Escalate

Escalate your issue if:

- **No response** within expected timeframe
- **Issue is blocking** production systems
- **Security concern** or data breach
- **Service outage** affecting multiple users

### How to Escalate

1. **Email escalation** to support@hackmamba.com
2. **Include "ESCALATION"** in subject line
3. **Reference original ticket** number
4. **Explain urgency** and business impact
